# 启动说明

## 1. 当前状态
- 后端现在可以不带数据库环境变量直接启动。
- 默认数据库已经改为本地 SQLite：`/root/Velo/backend/data/wiki.db`
- 默认 RAG embedding 已改为本地 Ollama：`nomic-embed-text:latest`
- 默认聊天模型：`qwen2.5:7b-instruct`
- 默认补全模型：`qwen2.5-coder:14b`
- 后端启动时会自动检查当前 RAG collection；如果 collection 为空，会把数据库里的现有文档自动重建索引。

## 2. 重启服务器后的标准启动顺序

### 2.0 先放好 API Key
如果后端要切到 DeepSeek，不要把密钥写死进代码，也不要放进前端。

正确位置：
```bash
/root/Velo/backend/.env
```

可以直接从示例文件复制：
```bash
cd /root/Velo/backend
cp .env.example .env
chmod 600 .env
```

然后把 `DEEPSEEK_API_KEY` 改成你的真实值。最小配置如下：
```env
LLM_PROVIDER=deepseek
DEEPSEEK_API_KEY=你的真实密钥
DEEPSEEK_API_BASE=https://api.deepseek.com/v1
CHAT_MODEL=deepseek-v4-flash
COMPLETION_MODEL=deepseek-v4-flash
```

说明：
- `.env` 已被 git 忽略，不会被正常提交。
- 密钥只允许出现在后端 `.env` 或系统环境变量里。
- 前端任何 `VITE_*` 变量都不能放真实密钥，因为会直接下发到浏览器。

### 2.1 启动 Ollama
```bash
cd /root/Velo
ollama serve
```

说明：
- 如果 Ollama 已经作为系统服务启动，这一步可以跳过。
- 需要确保 `ollama list` 里至少有这三个模型：
  - `qwen2.5-coder:14b`
  - `qwen2.5:7b-instruct`
  - `nomic-embed-text:latest`

### 2.2 启动后端
现在默认配置已经写进代码，直接执行即可：

```bash
cd /root/Velo/backend
/root/Velo/.venv/bin/python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

说明：
- 不再需要手动写 `POSTGRES_SERVER=sqlite`
- 不再需要手动写 `EMBEDDING_PROVIDER=huggingface`
- Redis 没启动也没关系，后端会自动降级到内存缓存
- 如果当前 RAG collection 是空的，后端启动时会自动把数据库中的文档重建索引

### 2.3 启动前端
```bash
cd /root/Velo/frontend
npm run dev -- --host 127.0.0.1 --port 5173
```

### 2.4 打开页面
```text
http://127.0.0.1:5173
```

后端健康检查：
```text
http://127.0.0.1:8000/health
```

## 3. 当前代码的默认行为
如果你什么环境变量都不传，后端默认就是：

- 数据库：SQLite
- 数据库文件：`/root/Velo/backend/data/wiki.db`
- Redis：优先连接本地 Redis，失败就降级内存缓存
- 聊天模型：`qwen2.5:7b-instruct`
- 补全模型：`qwen2.5-coder:14b`
- RAG embedding：`nomic-embed-text:latest`
- Ollama 地址：`http://127.0.0.1:11434/v1`

## 4. 启动后怎么自检
建议按这个顺序检查：

1. `ollama list`
2. 打开 `http://127.0.0.1:8000/health`，确认返回 `{"status":"ok"}`
3. 打开 `http://127.0.0.1:5173`
4. 创建或打开一篇文档
5. 在聊天里测试 RAG

## 5. 如果 8000 端口被占用
先检查：

```bash
ss -ltnp | grep ':8000 '
```

如果有旧进程占着，停掉它：

```bash
kill <PID>
```

或者临时改端口：

```bash
cd /root/Velo/backend
/root/Velo/.venv/bin/python -m uvicorn app.main:app --host 127.0.0.1 --port 8002
```

前端一起改代理：

```bash
cd /root/Velo/frontend
BACKEND_URL=http://127.0.0.1:8002 npm run dev -- --host 127.0.0.1 --port 5173
```

## 6. 如果你想后台运行
### 6.1 后台启动 Ollama
```bash
nohup ollama serve >/root/Velo/logs/ollama.log 2>&1 &
```

### 6.2 后台启动后端
```bash
cd /root/Velo/backend
nohup /root/Velo/.venv/bin/python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 >/root/Velo/logs/backend.log 2>&1 &
```

### 6.3 后台启动前端
```bash
cd /root/Velo/frontend
nohup npm run dev -- --host 127.0.0.1 --port 5173 >/root/Velo/logs/frontend.log 2>&1 &
```

### 6.4 查看日志
```bash
tail -f /root/Velo/logs/backend.log
tail -f /root/Velo/logs/frontend.log
tail -f /root/Velo/logs/ollama.log
```

## 7. 一句话结论
现在后端启动已经默认走本地 SQLite，RAG 也默认走本地 Ollama embedding。服务器重启后，按 `ollama -> backend -> frontend` 的顺序启动即可。
